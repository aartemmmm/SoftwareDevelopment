# Описание компонентов

## 1. Data Ingestion (Producers)

**Назначение:**  
Получение данных с биржевых источников.

**Функции:**
- Подключение к API (WebSocket / FIX)
- Нормализация данных
- Публикация сообщений в Kafka

**Цель:**  
Обеспечить стабильный и быстрый поток данных.

---

## 2. Apache Kafka (Message Broker)

**Назначение:**  
Центральный брокер сообщений.

**Функции:**
- Приём сообщений от продюсеров
- Хранение данных в топиках
- Распределение по партициям
- Репликация данных

**Цель:**  
Обеспечить масштабируемую и отказоустойчивую передачу данных.

---

## 3. Stream Processing (Flink / Spark Streaming)

**Назначение:**  
Обработка данных в реальном времени.

**Функции:**
- Агрегации (OHLC, VWAP)
- Расчёт индикаторов
- Обнаружение аномалий

**Цель:**  
Преобразование сырых данных в аналитические результаты.

---

## 4. Real-time Alerts Service

**Назначение:**  
Генерация уведомлений и сигналов.

**Функции:**
- Обработка событий
- Применение бизнес-правил
- Генерация алертов

**Цель:**  
Мгновенная реакция на изменения рынка.

---

## 5. Storage Layer

**Назначение:**  
Хранение данных.

**Варианты:**
- PostgreSQL (операционные данные)
- ClickHouse (аналитика)
- S3 / Hadoop (Data Lake)

**Функции:**
- Хранение истории
- Поддержка аналитических запросов

**Цель:**  
Обеспечить долговременное хранение и доступ к данным.

---

## 6. Backend API

**Назначение:**  
Предоставление данных клиентам.

**Функции:**
- REST / GraphQL API
- Доступ к данным и аналитике

**Цель:**  
Связь между системой и пользовательскими интерфейсами.

---

## 7. Frontend / Applications

**Назначение:**  
Визуализация данных.

**Функции:**
- Отображение графиков
- Дашборды
- Пользовательский интерфейс

**Цель:**  
Удобное представление информации пользователю.

---

## 8. Notification Service

**Назначение:**  
Доставка уведомлений.

**Функции:**
- Email
- SMS
- Push-уведомления

**Цель:**  
Оповещение пользователей о событиях в реальном времени.

---
